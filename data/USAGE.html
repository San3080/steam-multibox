Persiapan Awal (sekali saja)

1. Pasang prasyarat

  • Sandboxie (Classic atau Plus). Unduh dari github.com/sandboxie-plus. Instal, restart kalau diminta. Boleh di drive apa pun.
  • Steam terinstal di mana saja.
  • Python 3.10+. Cek: python --version.

2. Pasang dependensi tool

cd "D:\project tools\auto create profile sandboxie"
pip install -r requirements.txt

3. Pastikan service Sandboxie jalan

Sandboxie punya service (SbieSvc) yang harus aktif. Biasanya auto-start saat boot. Cek:

sc query SbieSvc

Kalau STATE: RUNNING → siap. Kalau tidak: buka SandMan sekali atau jalankan net start SbieSvc (admin).

Catatan: Setelah service jalan, kamu tidak perlu buka UI Sandboxie tiap pakai tool. Tool berkomunikasi langsung ke service.

Menyiapkan accounts.txt

  1. Buka folder data/ di samping app.py. Kalau belum ada, folder akan dibuat otomatis saat tool pertama kali dijalankan.
  2. Salin data/accounts.example.txt menjadi data/accounts.txt.
  3. Buka data/accounts.txt di Notepad / VS Code, isi dengan format satu akun per baris: 
    # Komentar diawali '#' diabaikan. Pemisah boleh ',' atau '|'.
andi_main,passwordku123
budi_alt|rahasia456
citra_fam,p@ssw0rd!special
  4. Simpan file dengan encoding UTF-8.
⚠️ Keamanan: File ini berisi password plaintext. Pastikan folder data/ tidak diakses orang lain (NTFS permission). File ini sudah otomatis masuk .gitignore.

Run Pertama

  1. Buka terminal di folder proyek, jalankan: 
    python app.py
    Jendela aplikasi terbuka, tema gelap.
  2. Lihat panel LOG di sidebar kiri-bawah. Akan muncul kira-kira: 
    HH:MM:SS  Sandboxie: pesan non-fatal di-silence global.
HH:MM:SS  N akun dimuat dari data/accounts.txt
HH:MM:SS  aplikasi siap.
    Kalau ada baris merah "Sandboxie/Steam belum terdeteksi" → lanjut ke langkah 3.
  3. Klik Pengaturan. Dialog terbuka. Cek 3 field path:
    • Folder Sandboxie — klik Auto Find (memindai semua drive); kalau gagal, klik Browse dan pilih folder yang berisi Start.exe & SbieIni.exe.
    • Path steam.exe — sama, Auto Find dulu, atau Browse ke steam.exe langsung.
    • File accounts.txt — default data/accounts.txt. Biarkan saja.
    Klik Simpan. Pengaturan tersimpan ke config.json.
  4. Pastikan kolom kanan menampilkan daftar akun dari accounts.txt. Tiap baris: ☑ checkbox · ● status · username · nama box · ✏️ Edit · 🗑 Hapus.
  5. Klik Run Semua. Kalau muncul popup "Steam host terdeteksi", klik Yes — tool akan matikan Steam asli supaya Steam di sandbox bisa start.
  6. Tunggu & perhatikan layar. Tiap akun:
    • Jendela Steam pop-up — tool isi form login otomatis (Remember me dicentang).
    • Kalau akun pakai Steam Guard via email/HP: prompt muncul → approve manual. Tool akan tetap menunggu (status: Tunggu 2FA).
    • Setelah Healthy, log menunjukkan "login terverifikasi (MostRecent): <user>" dan "RememberPassword=1 disetel".
    • Tool lakukan graceful shutdown 12 detik (Steam save token) → terminate → lanjut akun berikut.
  7. Setelah semua akun selesai, log: 
    HH:MM:SS  selesai: N sukses, M gagal.
    Akun gagal dicatat di data/fail.txt.
  8. Buka SandMan/SbieCtrl → klik kanan box (mis. Steam_andi_main) → Run Sandboxed → Run from Start Menu → Steam. Steam akan auto-login ke akun yang sesuai box, tanpa minta password lagi. 🎉

Aksi Per-Baris Box

Setiap baris akun di kolom kanan punya 2 tombol kecil di kanan badge status:

✏️ Edit Credential

  1. Klik ✏️ pada baris yang mau diubah.
  2. Dialog buka, field username + password sudah terisi (kalau mau ubah dua-duanya boleh, kalau cuma password ya cuma password).
  3. Klik Save. Popup konfirmasi: "Profile sandbox X akan di-clear lalu login ulang dengan credential baru. Lanjutkan?"
  4. Yes: tool jalankan urutan:
    1. Kalau username berubah → popup tambahan "Hapus box lama?" Yes/No.
    2. Update baris di accounts.txt.
    3. Wipe sesi Steam di box (loginusers.vdf + config.vdf + ssfn*).
    4. Reload daftar akun di UI.
    5. Run akun itu sendirian (login ulang dengan credential baru).
  5. Cancel di mana pun → tidak ada perubahan.

🗑 Hapus Box

  1. Klik 🗑 pada baris.
  2. Popup konfirmasi: "Hapus box 'Steam_X' beserta datanya?"
  3. Yes: tool terminate proses di box, hapus isi file (delete_sandbox), lalu hapus section di Sandboxie.ini. Box hilang dari Sandboxie UI.
Perbedaan Hapus vs Stop:
  • Stop = matikan proses, data box tetap (login session aman).
  • Hapus = matikan + hapus seluruh data box (login session hilang, akan login ulang dari nol saat Run berikutnya).

Discord Webhook (Opsional)

Tool bisa mengirim notifikasi ke channel Discord saat Run, login akun, edit credential, dan ringkasan akhir.

Setup

  1. Di Discord: Server Settings → Integrations → Webhooks → New Webhook.
  2. Pilih channel, beri nama (mis. "Steam Multi-Box"), klik Copy Webhook URL.
  3. Di tool: Pengaturan → field Discord webhook → paste URL.
  4. Klik tombol Test. Popup hijau "Pesan test terkirim" → cek channel Discord, harus muncul: 🔧 Test webhook dari Steam Multi-Box Launcher.
  5. Klik Simpan. URL tersimpan ke config.json.
Field kosong = notifikasi mati. Tidak ada side effect kalau kamu tidak mau pakai.

Pemakaian Harian

Workflow normal

  1. python app.py
  2. Pastikan Steam host mati (kalau jalan, tool akan kasih popup minta tutup).
  3. Klik Run Semua (atau Run Tercentang untuk subset).
  4. Approve 2FA kalau muncul.
  5. Setelah ringkasan "selesai", tutup tool atau biarkan untuk Run lagi nanti.

Menambah akun baru

  1. Edit data/accounts.txt, tambah baris baru user,pass.
  2. Di tool: klik Pengaturan → Simpan — ini akan reload daftar akun (atau restart app).
  3. Baris baru muncul di daftar box.
  4. Klik Run Tercentang dengan cuma akun baru yang dicentang (atau Run Semua kalau mau refresh sekalian).

Mengganti password akun yang sudah ada

  1. Klik ✏️ di baris akun itu.
  2. Edit field password, biarkan username sama.
  3. Save → Yes konfirmasi → tool wipe sesi + login ulang otomatis.

Mengganti username (rename) akun

  1. Klik ✏️.
  2. Ganti username (+ password kalau perlu). Save.
  3. Popup pertama: konfirmasi clear & relogin → Yes.
  4. Popup kedua: "Hapus box lama?"
    • Yes (rekomendasi): box lama dihapus bersih.
    • No: box lama dibiarkan, hapus manual di SandMan kalau perlu.
  5. Tool buat box baru (nama disesuaikan), login dengan credential baru.

Troubleshoot

GejalaDiagnosa & solusi
"Sandboxie/Steam belum terdeteksi" di log saat startup Klik Pengaturan → Auto Find. Kalau tetap kosong: Browse manual.
"Tidak ada akun yang dimuat" Cek data/accounts.txt ada & berisi minimal 1 baris valid (format user,pass atau user|pass). Encoding UTF-8.
Steam di sandbox muncul tapi minta login form kosong Hampir pasti Steam host masih jalan dan menerima command-line dari sandbox. Klik Tutup Steam Host dulu, lalu Run lagi.
"Too Many Retries" atau "Something went wrong while attempting to sign you in" Steam server rate-limit akun karena terlalu banyak percobaan login dalam waktu singkat. Tunggu 5-30 menit. Tool sudah mendeteksi pesan ini dan tidak retry lagi (status: LOGIN_FAILED).
Akun stuck di "Tunggu 2FA" Bukan bug — Steam Guard menunggu approve dari email/HP kamu. Approve dulu, controller akan lanjut sendiri.
Akun stuck di "Stuck splash" lalu retry berulang Steam tidak selesai loading dalam 40 detik (splash_timeout). Bisa karena koneksi lambat / Steam update besar. Naikkan splash_timeout di Pengaturan kalau sering kena.
Manual reopen box minta login lagi Cek config.json: login_method harus "ui" (default). Kalau masih "cmdline", tool tidak centang Remember me. Auto-migration sudah ada, tapi kalau diubah manual, balik ke "ui".
Edit credential gagal "akun tidak ditemukan" Username di accounts.txt sudah berubah sejak tool dibuka. Klik Pengaturan → Simpan untuk reload, lalu coba edit lagi.
Hapus box gagal dengan exit 1 Coba dulu klik Stop / Terminate untuk pastikan tidak ada proses yang aktif di box. Lalu retry hapus. Kalau tetap gagal, hapus manual lewat SandMan (klik kanan box → Remove Sandbox).
Discord Test gagal Popup error sekarang menampilkan kode HTTP / pesan eksak.
  • HTTP 401 → webhook token salah / di-revoke.
  • HTTP 404 → webhook URL sudah dihapus.
  • Tidak bisa connect → firewall / koneksi internet.
  • URL harus diawali http(s):// → format URL salah.
Tool lemot / RAM penuh saat Run 7+ akun Wajar — tiap Steam client ≈ 250-400 MB RAM. Pakai Auto-terminate setelah login sukses supaya box mati setelah token tersimpan. Atau pakai Run Tercentang dengan batch lebih kecil.

Referensi Field config.json

Biasanya cukup pakai dialog Pengaturan, tapi untuk advanced user:

FieldDefaultPenjelasan
sandboxie_dirauto-detect Folder instalasi Sandboxie (berisi Start.exe & SbieIni.exe).
steam_exeauto-detect Path absolut ke steam.exe.
accounts_filedata/accounts.txt Path ke file daftar akun.
box_prefixSteam_ Nama box = prefix + username.
stagger_seconds8 Jeda antar akun setelah satu selesai diproses.
login_methodui ui = isi form login UI (Remember me, persistent). cmdline = pakai steam.exe -login (cepat tapi token tidak tersimpan untuk reopen).
auto_terminate_on_successfalse Centang sidebar untuk men-set true: tiap box di-terminate setelah login sukses.
max_retries3 Berapa kali ulang jika stuck/mismatch. Total attempt = max_retries + 1.
retry_check_delay25 Jeda awal (detik) sebelum mulai polling status box.
poll_interval5 Jeda antar polling status box (detik). Lebih kecil = stop lebih responsif.
splash_timeout40 Berapa detik nungguin Steam splash sebelum dianggap stuck.
terminate_grace_seconds12 Jeda setelah steam.exe -shutdown dipanggil, sebelum hard terminate. Beri Steam waktu menulis encrypted token.
discord_webhook_url"" URL Discord webhook. Kosong = notif mati.
