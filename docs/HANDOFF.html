3. Alur Hidup Sebuah Akun (Run)

process_account(account):
    set_status: LAUNCHING
    driver.ensure_box(box)              # create box + silence noisy SBIE

    for attempt in 1..max_retries+1:
        prime_clean_steam_config(root)  # wipe inherited host vdf/config/ssfn
        driver.launch(account, box)     # ui mode: launch polos, NO -login

        _await_login(account, box):
            wait retry_check_delay
            for _ in range(_MAX_POLLS_PER_ATTEMPT):
                state = driver.poll(box, elapsed, expected_username=account.username)
                # state via VDF fast-path + title fallback

                if HEALTHY:           return "ok"
                if LOGIN_FAILED:      return "fail"  # no retry, password salah/rate-limit
                if STUCK_RETRY|SPLASH:return "stuck" # akan retry
                if PICKER:            click_add() (once)
                if LOGIN_FORM:        fill_login (once, with Remember me)
                # WAITING_2FA | LAUNCHING | UNKNOWN -> keep watching
                wait poll_interval

        if outcome == "ok":
            _check_login_match():
                verify_login(box, user)
                if MATCH:      enable_remember_password (RememberPassword=1)
                if MISMATCH:   wipe_steam_session + continue (retry)
            return success

        if outcome == "fail":   safe_terminate + return False
        if outcome == "stuck":  safe_terminate + retry next iteration

    return False, "login terus ke akun salah atau stuck setelah N retry"

# After all accounts:
write_failures(fail_path, failures)
notify Discord (best-effort)

4. Strategi Deteksi Healthy / Mismatch

Healthy detection di steam_ui.snapshot_box punya dua jalur:

4.1 VDF fast-path (primary)

  1. Cari folder box di disk (find_box_root mengecek C:\Sandbox\<user>\<box> + variannya).
  2. Buka drive\C\Program Files (x86)\Steam\config\loginusers.vdf di box.
  3. Parse semua AccountName. Jika expected_username ada → tandai main_window=True → classify HEALTHY.

Jika VDF belum tertulis Steam (early polling), fast-path miss → fallback ke title scraping.

4.2 Title scraping (fallback)

Enumerate semua jendela desktop via pywinauto.Desktop("uia").windows(), ambil title. Filter [#] marker (Sandboxie Classic) atau [<box>] prefix; kalau tidak ada marker (Sandboxie Plus), analisis semua title (relaxed mode — dengan asumsi Steam host sudah dimatikan).

Marker yang dideteksi:

  • "library" / "store page" / "friends list" / "your library" → main window post-login.
  • "sign in to steam" → LOGIN_FORM.
  • "who's playing" → ACCOUNT_PICKER.
  • "steam guard" → WAITING_2FA.
  • "please check your password" / "too many retries" / "too many sign in failures" / "something went wrong while attempting to sign" → LOGIN_FAILED.
  • "retry" / "could not connect" → STUCK_RETRY.

4.3 Verifikasi MostRecent

Setelah HEALTHY, verify_login parsing ulang vdf dengan strategi:

  1. Cari blok user yang berisi "MostRecent" "1". Ambil AccountName-nya. Itu yang BENAR-BENAR sedang aktif.
  2. Bandingkan ke expected. Match → lanjut. Mismatch → wipe + retry.
  3. Kalau tidak ada MostRecent (vdf masih segar), fallback ke "ada di file".

5. Keputusan Desain Penting

KeputusanAlasan
Default login_method=ui, bukan cmdline cmdline (steam.exe -login) tidak save encrypted token. UI form + Remember me memastikan manual reopen auto-login.
Wipe Steam state SEBELUM tiap launch attempt Sandboxie copy-on-read salin loginusers.vdf & config.vdf host ke box. Tanpa wipe, box mewarisi 4-5 akun host yang tidak relevan.
Hard-kill host Steam sebelum Run Steam single-instance: kalau host jalan, launch dari sandbox diteruskan ke host — bukan login ke akun box.
Graceful shutdown 12s sebelum hard terminate Tanpa graceful, Steam tidak sempat tulis encrypted token ke config.vdf. Akibatnya manual reopen minta password lagi.
Edit Sandboxie.ini langsung untuk delete box SbieIni delete exit 1 di Sandboxie Plus. Edit text langsung + SbieIni reload bekerja universal.
fill_login & click_add hanya SEKALI per attempt Sebelumnya tool isi form tiap polling → Steam rate-limit cepat (Too Many Retries).
LOGIN_FAILED bypass retry loop Kalau Steam menolak (password salah / rate-limit), retry membabi-buta malah membuat akun terkunci. Lebih baik langsung fail dan masuk fail.txt.
SBIE silence di-set per-box DAN di [GlobalSettings] Popup "Messages from Sandboxie" nge-blok automasi. Global setting memastikan box baru pun terpengaruh tanpa harus reconfigure.
Run bergiliran (sequential), bukan paralel Lebih ramah ke Steam server (anti rate-limit), lebih mudah verifikasi, dan jendela Steam tidak menumpuk semua sekaligus.
Folder data/ untuk semua user file Mudah backup, mudah .gitignore, jelas terpisah dari kode.
Discord webhook best-effort total Kalau Discord/internet mati, run tidak boleh terganggu. Semua call tertangkap di-swallow.

6. Batasan & Known Issues

By design (tidak akan diubah):
  • Tidak men-generate kode 2FA. User approve manual.
  • Tidak menyimpan shared secret / maFile.
  • Tidak membuat akun Steam.
  • Plaintext password di data/accounts.txt — user diberi peringatan dan file di-gitignore. Untuk enkripsi DPAPI, lihat ide di bagian 10.
Diketahui (workaround tersedia):
  • Steam Guard email/HP: tool tidak meng-otomasi prompt 2FA. User harus klik approve sendiri saat status box "Tunggu 2FA".
  • UI selector Steam berubah: kalau Valve update bentuk form login, fill_login_form bisa miss. Selector ada di src/steam_ui.py, mudah disesuaikan. Fallback: timeout + retry akan mendeteksi sebagai stuck.
  • Box folder di lokasi non-standar: kalau Sandboxie FileRootPath di-set ke drive lain (mis. E:\Sandbox), find_box_root hanya mengecek 3 kandidat hard-coded. Solusi: parse FileRootPath dari ini sebagai TODO future.
  • Edit Credential gagal saat username tidak ada: terjadi jika accounts.txt sudah diubah lewat editor luar sejak tool dibuka. Reload via Pengaturan-Simpan.

8. Catatan Operasional

Encoding

  • config.json, accounts.txt, fail.txt — UTF-8.
  • Sandboxie.ini — biasanya UTF-16 LE BOM. Handler _read_ini deteksi BOM otomatis dan write-back dengan encoding yang sama.
  • loginusers.vdf, config.vdf — UTF-8 (Steam).

Lokasi default

  • Folder data: data/ di samping app.py.
  • Folder box: C:\Sandbox\<WindowsUser>\Steam_<username>\.
  • Sandboxie.ini: C:\Windows\Sandboxie.ini atau registry-defined.

Permission

  • Tool berjalan sebagai user biasa. Sandboxie service yang elevasinya admin.
  • Menulis C:\Windows\Sandboxie.ini kadang butuh elevasi pada konfigurasi tertentu. Jika delete gagal dengan PermissionError, jalankan tool as admin (jarang dibutuhkan untuk Sandboxie Plus default).

Threading

  • GUI di main thread.
  • Controller (run loop) di daemon thread, dimulai oleh Controller.run_async.
  • Komunikasi: Controller.updates queue + status_cb callback yang dispatch via root.after(0, ...) ke GUI thread.
  • Stop signal: threading.Event dengan stop_event.wait(timeout) agar sleep bisa di-interrupt.

Webhook / network

  • Discord call menggunakan stdlib urllib.request. Tidak ada dependency baru.
  • Timeout default 5 detik. Tidak ada retry — best-effort.
  • Pesan dipotong di 1900 karakter (batas Discord 2000).
